Week 3

  • Last week: I researched the scikit-learn library, learning about some of the applications that it has for use in my Content-Based-Recommendation-System project. At this stage I focused on utilizing scikit to vectorize the keyword data associated with movies to be used later for similarity comparisons. I was able to import the data directly from Kaggle into a data frame, normalize keyword formatting, address data type inconsistencies, and append a column to the set containing the preprocessed data in a space separated string of keywords. Lastly, I used CountVectorizer to transform the dataset into a sparse matrix of keywords contained for each movie.

  • This week: I plan to design the SQLite schema structure and to create the CRUD operations necessary for its operation. In the database I plan to store the metadata of the movies along with the keyword strings, as storing entire matrix rows is not feasible given their size and sparsity. I also plan to store a user watch history in the database.

  • Obstacles: The main challenge will be time management. While I have experience with designing and implementing relational databases, as well as using libraries such as pymongo, but I am new to using sqlite3 and will need to familiarize myself with its syntax and behaviors.
